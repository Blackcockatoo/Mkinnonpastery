# Resume Maintenance Guide

This folder keeps public-facing and private resume materials aligned while preserving confidentiality where required.

## File Purpose
- **Public-facing resume:** `resume/mark-mckinnon-resume.md`  
  Uses anonymized employers for website/portfolio and broad sharing.
- **Public-facing website copy:** `resume/web-bio.md`  
  Includes short bio blocks and scannable specialties for CMS use.
- **Private/full resume:** create and maintain `resume/mark-mckinnon-resume-private.md` (not for public publishing).  
  This version can include named employers, full chronology, and private contact details.

## What Must Stay Identical Across Public and Private Versions
Keep these sections aligned in meaning and chronology:
1. Professional summary
2. Core skills
3. Training/qualifications

## Update Checklist (every revision)
1. Confirm role titles and date ranges are accurate and current.
2. Keep positioning consistent: **Pastry Chef | Cake & Pastry Specialist | Baker**.
3. Verify credential wording against the certificate.
4. Preserve anonymized employer labels in public files.
5. Remove or complete placeholders before publishing where required (`[add]`, location variant, portfolio link, ABN field).
6. Validate service offer wording (subcontract, catering, recipe development) reflects current business scope.

## Qualification Wording Rule
Always preserve the qualification title exactly as printed on the certificate in official records. If needed for employer readability, add a modern descriptor in parentheses without changing the original credential wording.
